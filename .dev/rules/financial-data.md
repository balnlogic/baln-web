# 금융 데이터 정확성 규칙 (baln-web)

> 근거: 금융 데이터에서 부동소수점 오류나 통화 포맷 불일치는 사용자에게 잘못된 금액 정보를 보여주어 심각한 신뢰 손상을 일으킨다.

- [ ] 금액 계산에 JavaScript의 부동소수점 연산을 직접 사용하지 않는가? (`0.1 + 0.2 !== 0.3` 문제 — `decimal.js` 또는 정수 변환 후 계산)
- [ ] 금액은 항상 원 단위 정수(또는 백분의 1원 단위 정수)로 저장하고, 표시 시에만 포맷 변환하는가?
- [ ] 통화 포맷은 `Intl.NumberFormat('ko-KR', { style: 'currency', currency: 'KRW' })`를 사용하는가?
- [ ] 퍼센트(수익률, 이자율)는 소수점 2자리까지만 표시하는가? (반올림 기준 명시)
- [ ] 금액 관련 API 응답값은 `number` 타입으로 단언하기 전에 `isFinite()` 검증을 하는가?
- [ ] 음수 금액(손실, 차감)이 UI에서 빨간색 등 명확한 시각적 구분으로 표시되는가?
- [ ] 금액 입력 필드에서 사용자가 문자, 특수문자를 입력할 수 없도록 validation이 있는가?
- [ ] 큰 금액(억 단위 이상)의 경우 `만`, `억` 단위 약식 표기와 전체 금액 툴팁이 함께 제공되는가?
