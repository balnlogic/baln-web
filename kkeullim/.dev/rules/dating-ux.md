# 데이팅 UX 규칙 (kkeullim — 골드스푼 모델)

> 근거: 골드스푼 모델(매치 시 전화번호 공개)은 개인정보 노출이 핵심 기능이므로 양방향 동의와 단계별 공개 로직을 반드시 검증해야 한다.

- [ ] 전화번호 공개는 **양방향 매치(mutual like) 확정 후**에만 이루어지는가?
- [ ] 전화번호를 포함한 민감 정보는 매치 성립 전까지 클라이언트로 절대 전송되지 않는가?
- [ ] 매치 취소/언매치 시 상대방의 전화번호가 즉시 숨겨지는가? (로컬 상태 + DB 동기화)
- [ ] 좋아요/싫어요 스와이프 액션이 중복 실행되지 않도록 debounce 처리가 되어 있는가?
- [ ] 프로필 카드 스택이 빈 경우(`empty state`) 적절한 UI 안내를 보여주는가?
- [ ] 골드스푼 등급 표시 로직이 실제 프리미엄 사용자 여부와 일치하는가? (Zustand 상태 vs Supabase DB 동기화)
- [ ] 매치 알림(Push Notification)이 앱이 백그라운드일 때도 정상 수신되는가?
- [ ] 더미 프로필과 실제 사용자 프로필이 섞이지 않도록 `is_dummy` 플래그로 구분하는가?
- [ ] 신고/차단 기능이 존재하고, 차단된 사용자의 프로필이 즉시 스택에서 제거되는가?
- [ ] 다크모드(Primary `#FF2D55`)에서 가독성 대비 비율이 WCAG AA 기준을 만족하는가?
